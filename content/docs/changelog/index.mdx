# Changelog

Semua perubahan penting pada go-migration didokumentasikan di halaman ini. Format mengikuti [Keep a Changelog](https://keepachangelog.com/), dan proyek ini menggunakan [Semantic Versioning](https://semver.org/).

## v1.5.0 (2026-03-20)

### Added

- Method `RawExec(statements ...string)` pada `schema.Builder` untuk mengeksekusi raw SQL langsung — berguna untuk materialized views, custom DDL, atau operasi yang tidak di-cover oleh fluent API
- Test baru di `interpolate_test.go` untuk validasi auto-unquote pada `InterpolateEnv`

### Changed

- `InterpolateEnv` (`interpolate.go`) sekarang otomatis melakukan unquote `"${VAR}"` di file JSON ketika value environment variable berupa numerik, boolean (`true`/`false`), atau `null`. Contoh: `"port": "${DB_PORT}"` dengan `DB_PORT=5432` menghasilkan `"port": 5432` (bukan string `"5432"`)

**Tidak ada breaking changes** — perubahan `InterpolateEnv` hanya berlaku untuk file JSON dan hanya pada value yang valid sebagai numerik, boolean, atau null. Value string tetap mempertahankan tanda kutip seperti sebelumnya.

## v1.4.2 (2025-07-15)

### Added

- Custom command router (`router.go`) menggantikan cobra, menggunakan `map[string]CommandHandler` dan package `flag` dari stdlib
- File baru: `router.go`, `router_property_test.go` (3 property test: routing, unknown command, flag parsing), dan `init_config.json.tmpl`
- Property test baru: Property 5 (JSON round-trip), Property 6 (cross-format equivalence), Property 7 (env var interpolation in JSON)

### Changed

- Format konfigurasi default berubah dari `go-migration.yaml` ke `migration.json` — command `init` sekarang menghasilkan `migration.json`
- Semua 14 file command di `pkg/cli/commands/` menggunakan signature `CommandHandler` baru dengan parameter `io.Reader`/`io.Writer` (bukan `*cobra.Command`)
- Semua assertion testify (~50 file test) diganti dengan `if` + `t.Errorf`/`t.Fatalf` dari testing stdlib
- Total dependensi berkurang dari 13 menjadi 7 paket
- Dependensi tersisa: go-sqlmock, go-sql-driver/mysql, lib/pq, go-sqlite3, yaml.v3, pgregory.net/rapid, edwards25519

### Deprecated

- Format konfigurasi YAML (`go-migration.yaml`) — masih didukung tetapi menampilkan peringatan deprecation di console saat dimuat

### Removed

- 6 dependensi dihapus: spf13/cobra, spf13/pflag, inconshreveable/mousetrap, stretchr/testify, davecgh/go-spew, pmezard/go-difflib
- File `root.go` dihapus (fungsionalitas dipindahkan ke Router)

**Tidak ada breaking changes** — public API (`migrator.New()`, `migrator.Run()`, `Migration` interface) tetap sama. YAML config masih didukung, hanya muncul deprecation warning.

## v1.4.0 (2025-06-15)

### Added

- Command `go-migration init` untuk scaffolding proyek baru (membuat direktori, file konfigurasi, dan entry point CLI)
- Format penamaan migrasi baru bergaya Laravel: `YYYY_MM_DD_HHMMSS_RRRR_description.go` dengan segmen random 4-digit
- Template migrasi yang dihasilkan `make:migration` sekarang otomatis menyertakan fungsi `init()` dengan `migrator.AutoRegister()`

### Changed

- Nama file konfigurasi default berubah dari `migration.yaml` menjadi `go-migration.yaml`
- Struktur konfigurasi baru mendukung field `default` dan `connections` untuk multiple database connections
- Direktori default berubah ke `database/migrations`, `database/seeders`, `database/factories` (user dengan config `migration_dir: "migrations"` yang sudah ada tetap berjalan normal)
- Format penamaan lama (`YYYYMMDDHHMMSS_description.go`) masih didukung untuk backward compatibility

## v1.3.0 (2025-06-15)

### Added

- Dukungan `--tag` flag pada command `db:seed` untuk menjalankan seeder berdasarkan tag tertentu
- Interface `TaggedSeeder` untuk menandai seeder dengan tag kategori
- Interface `RollbackableSeeder` dan command `db:seed:rollback` untuk rollback seeder
- Command `db:seed:truncate` untuk menghapus semua baris dari tabel tertentu
- Dukungan `--force` flag pada `migrate:reset` untuk melewati konfirmasi
- Dukungan `--seed` flag pada `migrate` dan `migrate:refresh` untuk menjalankan seeder setelah migrasi

### Changed

- `migrate:fresh` sekarang mendukung flag `--force` dan `--seed` secara bersamaan
- Peningkatan performa auto-discovery untuk migrasi dan seeder pada proyek besar
- Dokumentasi diperbarui untuk mencerminkan semua perubahan API v1.3.0

### Fixed

- Perbaikan urutan eksekusi rollback yang tidak konsisten pada batch dengan banyak migrasi
- Perbaikan error handling pada `AfterMigrate` hook yang menyebabkan panic jika callback nil
- Perbaikan `migrate:status` yang tidak menampilkan migrasi pending dengan benar pada SQLite

## v1.2.0 (2024-09-01)

### Added

- `migrator.Run()` — all-in-one CLI runner yang menangani config loading, koneksi database, auto-discovery, dan command dispatch dalam satu panggilan
- Smart command classification — command seperti `version`, `help`, dan `make:*` melewati setup database
- `migrator.AutoRegister()` untuk registrasi migrasi otomatis melalui fungsi `init()`
- `WithAutoDiscover()` option untuk auto-discovery migrasi dan seeder yang terdaftar
- Dukungan interpolasi environment variable `${VAR_NAME}` di file konfigurasi YAML
- Grammar auto-resolution dari field `driver` di file konfigurasi

### Changed

- Peningkatan output CLI dengan format tabel yang lebih rapi pada `migrate:status`
- Dokumentasi Quick Start diperbarui untuk merekomendasikan `migrator.Run()` sebagai pendekatan utama

### Fixed

- Perbaikan koneksi database yang tidak ditutup dengan benar saat menggunakan `migrator.Run()`
- Perbaikan `make:migration` yang menghasilkan timestamp duplikat pada pembuatan file berurutan cepat
